client::send

Asynchronously send data to entire quorum and return result in a future. 

std::future< std::string > send(
    std::string const & byte_array,
    uint16_t retries = 10);
Parameters

byte_array

Data to sent. Binary-safe.

retries

Amount of times to retry failed operations

Return Value

Future to the result

Remarks

If retries is larger than 1, automatically retries the operation. This will result in future.get () taking longer to get a result, but it can greatly reduce the amount of exceptions exposed to the code making use of this client; situations like leadership failover can cause temporary errors to be thrown, which usually resolve themselves within a second.

While this function by itself does not throw any exceptions due to its asynchronous nature, errors can occur while processing the request. If this is the case, the exception will be stores inside the future, and any call to future.get () will throw this exception.
