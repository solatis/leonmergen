exception::inconsistent_response

Thrown when the callback function of multiple server instances do not reply with the same output for the same input. 

class inconsistent_response :
  public exception::exception

As described in the documentation of server, it is a requirement of the callback function of a server to always return the exact same response for similar commands.

In other words, consider this example:

  • server 1 replies to paxos message 123 with response "foo"
  • server 2 replies to paxos message 123 with response "bar"
  • since the responses do not match, an exception will be thrown.
